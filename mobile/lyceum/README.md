# Lyceum — Android app (Flutter)

Native Android client for the Lyceum ebook server (LYCM-700). Mirrors the web
app's design language and talks to the same unauthenticated core API. Built to
match the Argosy mobile stack (Riverpod + go_router + a design-token port).

## Architecture

**Hybrid, by deliberate choice.** The library, settings and server-config
screens are native Flutter. The **reader is a `webview_flutter`** loading the
backend's own epub.js reader page (`{server}/reader/:id`).

Why: Lyceum syncs reading position via **EPUB-CFI**, and those CFI strings are
generated by epub.js. A native Dart renderer would produce incompatible CFIs and
break cross-device resume — sync is the core of the product. Loading the
backend-served reader keeps epub.js + CFI + `/sync` **byte-identical** and
same-origin with the API (the backend deep-links fall through to `index.html`,
see `cmd/lyceum/main.go`), so there is zero reimplementation on the sync path.

## Stack

- **Flutter 3.44.2 / Dart 3.12.2** (pinned; see `~/dev-tools`)
- **flutter_riverpod** — state + DI (`Notifier` / `AsyncNotifier`)
- **go_router** — 3 routes: `/` (library), `/reader/:id`, `/settings`
- **http** — typed API client (`lib/api/client.dart`)
- **shared_preferences** — server URL, device id, theme, reading font
- **webview_flutter** — the reader surface
- **file_selector** — EPUB upload picker

## Layout

```
  features/

```

## Develop

```sh
source ~/dev-tools/env.sh        # JAVA_HOME, ANDROID_HOME, flutter+sdk on PATH
cd mobile/lyceum
flutter pub get
flutter analyze                  # lint gate
flutter test                     # unit tests (models + normalizeServerUrl)

# Run on a connected device, optionally pre-seeding the server URL:
flutter run --dart-define=LYCEUM_BASE_URL=http://10.0.0.20:8080
```

On first launch (no `--dart-define`) the app shows a connect prompt — enter the
server URL, **Test** (`GET /healthz`), **Save**.

## Build

```sh
flutter build apk --debug        # → build/app/outputs/flutter-apk/app-debug.apk
flutter build apk --release      # → app-release.apk (sideloadable)
```

### Signing

Release signing reads `android/key.properties` (gitignored); when absent it
falls back to the debug key so `--release` still builds. To sign for real,
create `android/key.properties`:

```properties
storeFile=/absolute/path/upload-keystore.jks
storePassword=…
keyAlias=upload
keyPassword=…
```

R8/resource-shrinking is intentionally **off** for release (reflection-driven
plugin startup breaks under full R8 for a self-hosted client — cf. Argosy
ARGY-114).
